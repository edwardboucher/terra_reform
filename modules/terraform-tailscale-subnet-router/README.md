# Tailscale Subnet Router Terraform Module

This module deploys a Tailscale Subnet Router on AWS using an Auto Scaling Group (min=1) for automatic instance recovery. It installs and configures Tailscale to advertise specified subnet CIDRs into your tailnet.

## Requirements

- Terraform >= 1.2.0
- AWS account
- Tailscale API key (generated at https://login.tailscale.com/admin/settings/keys)

## Networking requirements

### Public subnet
The instance needs outbound internet access to reach Tailscale's coordination servers (`controlplane.tailscale.com`). In a public subnet, set `public_ip_enabled = true`.

### Private subnet
When deploying in a private subnet, a **NAT gateway** must exist in the VPC so the instance can reach Tailscale coordination servers. Set `public_ip_enabled = false` (the default).

## SSH vs SSM access

By default, `enable_ssm = true` creates an IAM role with `AmazonSSMManagedInstanceCore` attached, allowing access via AWS Systems Manager Session Manager ŌĆö no open port 22, no public IP required.

To enable traditional SSH instead (or in addition):
- Set `enable_ssh = true` and provide `ssh_allowed_cidr` (e.g. your public IP as `"203.0.113.5/32"`)
- Provide an SSH key via `ssh_key_name`

## ACL auto-approval

The bundled ACL template (`tailscale_acl.json.tpl`) configures auto-approval for routes advertised by nodes tagged with `var.tailscale_tag`. Set `refresh_tailscale_main_acl = true` to apply it. To use a custom ACL, pass JSON content via `tailscale_acl_content`.

## Breaking changes from previous versions

- **`tailscale_auth_key` renamed to `tailscale_api_key`** ŌĆö this is the Tailscale API key for the provider, not a node auth key (which is auto-generated by the module).
- **`aws_instance` replaced by Auto Scaling Group** ŌĆö outputs `tailscale_router_id` and `tailscale_router_public_ip` have been removed. Use `autoscaling_group_name`, `launch_template_id`, and `security_group_id` instead.

## Usage

```hcl
module "tailscale_subnet_router" {
  source = "github.com/edwardboucher/terra_reform/modules/terraform-tailscale-subnet-router"

  # Required
  ami_id              = "ami-026ebd4cfe2c043b2"   # RHEL 9 AMI
  ts_router_subnet_id = module.vpc_base.private_subnet_ids[0]
  tailscale_api_key   = var.tailscale_api_key
  tailscale_net       = "example.ts.net"
  advertise_routes    = [var.vpc_cidr]

  # Optional ŌĆö OS and instance
  os_distro           = "rhel"                    # or "ubuntu" / "debian"
  instance_type       = "t3.micro"
  public_ip_enabled   = false                     # true if public subnet, false if private + NAT

  # Optional ŌĆö access
  enable_ssm          = true                      # creates IAM role with SSM policy automatically
  enable_ssh          = false
  ssh_key_name        = "my-ssh-key"

  # Optional ŌĆö RHEL subscription (leave empty for Ubuntu/Debian or unregistered RHEL)
  rh_username         = var.rh_username
  rh_password         = var.rh_password

  # Optional ŌĆö Tailscale ACL
  tailscale_tag              = "vpc-peering"
  refresh_tailscale_main_acl = true

  # Optional ŌĆö tags
  tags = {
    Environment = "production"
    Team        = "platform"
  }
}
```

## Inputs

| Name | Description | Type | Default | Required |
|------|-------------|------|---------|----------|
| `ami_id` | AMI ID for the instance | `string` | ŌĆö | yes |
| `ts_router_subnet_id` | Subnet ID for the router | `string` | ŌĆö | yes |
| `tailscale_api_key` | Tailscale API key for the provider | `string` | ŌĆö | yes |
| `tailscale_net` | Tailscale network name (e.g. `example.ts.net`) | `string` | ŌĆö | yes |
| `advertise_routes` | CIDRs to advertise into the tailnet | `list(string)` | `["172.31.0.0/16"]` | no |
| `os_distro` | OS family: `rhel`, `ubuntu`, or `debian` | `string` | `"rhel"` | no |
| `instance_type` | EC2 instance type | `string` | `"t3.micro"` | no |
| `public_ip_enabled` | Assign a public IP to the instance | `bool` | `false` | no |
| `enable_ssm` | Create IAM role for SSM Session Manager access | `bool` | `true` | no |
| `iam_role` | Existing IAM role name to use (overrides SSM auto-role) | `string` | `""` | no |
| `enable_ssh` | Add SSH ingress rule to the security group | `bool` | `true` | no |
| `ssh_allowed_cidr` | CIDR allowed for SSH ingress | `string` | `""` | no |
| `ssh_key_name` | EC2 key pair name | `string` | `"linux-key-pair-tailscale"` | no |
| `tailscale_tag` | Tailscale ACL tag for the node | `string` | `"vpc-peering"` | no |
| `tailscale_acl_content` | Custom ACL JSON (overrides bundled template) | `string` | `""` | no |
| `refresh_tailscale_main_acl` | Apply the ACL to the tailnet | `bool` | `false` | no |
| `rh_username` | Red Hat subscription username (RHEL only) | `string` | `""` | no |
| `rh_password` | Red Hat subscription password (RHEL only) | `string` | `""` | no |
| `tags` | Tags to apply to all resources | `map(string)` | `{}` | no |

## Outputs

| Name | Description |
|------|-------------|
| `autoscaling_group_name` | Name of the Auto Scaling Group |
| `launch_template_id` | ID of the Launch Template |
| `security_group_id` | ID of the security group (use to allow traffic from the router) |
